## Feature Comparison

## When to Use Zoi

Zoi excels in specific scenarios:

### ✅ Use Zoi When:

1. **Cross-Platform Development**
   - Working on projects across Linux, macOS, and Windows
   - Sharing development environments with teammates on different OSes
   - Building portable tools that work everywhere

2. **User-Level Package Management**
   - You don't have sudo/admin access
   - You want to avoid polluting system directories
   - You need different versions than system packages

3. **Multi-Language Projects**
   - Your project uses tools from multiple ecosystems (Node.js, Python, Rust, etc.)
   - You want a single workflow to manage all dependencies
   - You need both system and language-specific tools

4. **Environment Reproducibility**
   - You need to recreate exact development environments
   - You want version pinning across all tools
   - You need to rollback to previous configurations

5. **Custom Package Distribution**
   - You want to distribute internal tools to your team
   - You need a simple way to package and share binaries
   - You want Git-based package hosting

6. **Integration with Existing Managers**
   - You want to complement apt/brew/pacman, not replace them
   - You need to manage dependencies across multiple package managers
   - You want a unified interface for different backends

### ❌ Use Alternatives When:

1. **System Package Management**
   - **Use apt/pacman/dnf** for:
     - System-critical packages and libraries
     - OS-level components and services
     - System-wide installations requiring deep integration

2. **macOS Native Experience**
   - **Use Homebrew** for:
     - macOS-specific optimizations
     - Casks for GUI applications
     - Formula with deep macOS integration

3. **Reproducible System Configurations**
   - **Use Nix** for:
     - Declarative system configuration
     - Isolated development environments
     - Functional package management philosophy

4. **Language-Specific Development**
   - **Use npm/cargo/pip/go** for:
     - Library dependencies in your code
     - Language-specific development tools
     - Package.json/Cargo.toml managed dependencies

5. **Containerized Environments**
   - **Use Docker** for:
     - Production deployments
     - Complete environment isolation
     - Multi-service orchestration

## Integration Stories

Zoi is designed to work alongside other package managers, not replace them.

### With System Package Managers

```sh
# System packages for libraries
sudo apt install libssl-dev build-essential

# User tools with Zoi
zoi install @zillowe/hello
zoi install @core/ripgrep
```

### With Language Package Managers

```yaml
# zoi.yaml - Project environment
packages:
  - name: node
    check: node --version
  - name: rust  
    check: cargo --version
    
dependencies:
  # Zoi manages the runtime tools
  - zoi:ripgrep
  - zoi:fd
  
  # npm manages JavaScript libraries
  - npm:typescript
  - npm:eslint
  
  # cargo manages Rust tools
  - cargo:cargo-watch
  - cargo:cargo-audit
```

Zoi can declare dependencies for 40+ package managers:

```lua
-- In a package definition
dependencies = {
  runtime = {
    "apt:libssl-dev",      -- Debian/Ubuntu
    "pacman:openssl",      -- Arch Linux  
    "brew:openssl@3",      -- macOS
    "npm:typescript",      -- Node.js
    "pip:requests",        -- Python
    "cargo:serde",         -- Rust
    "zoi:another-package", -- Zoi packages
  }
}
```

### With Homebrew on macOS

```sh
# Use Homebrew for macOS-specific apps
brew install --cask visual-studio-code
brew install --cask docker

# Use Zoi for cross-platform CLI tools
zoi install @core/jq
zoi install @main/gh
```

### With Nix for Development Shells

```nix
# shell.nix
{ pkgs ? import <nixpkgs> {} }:

pkgs.mkShell {
  buildInputs = [
    pkgs.git
    pkgs.gnumake
  ];
  
  shellHook = ''
    # Use Zoi for project-specific tools
    if ! command -v zoi &> /dev/null; then
      curl -fsSL https://zillowe.pages.dev/scripts/zoi/install.sh | bash
    fi
    zoi env dev
  '';
}
```

## Philosophy Differences

### Zoi: Integration & Flexibility

- Works with existing package managers
- Dynamic Lua-based package definitions
- User-focused, sudo-free by default
- Git-based, decentralized registries
- Cross-platform from the ground up

### apt/pacman: System Integrity

- Manages system-critical packages
- Strict versioning for stability
- Deep OS integration
- Centralized, curated repositories

### Homebrew: macOS Native

- Optimized for macOS experience
- Formula-based package definitions
- Strong community ecosystem
- User-level installations

### Nix: Functional Purity

- Reproducible builds by design
- Isolated package environments
- Declarative configuration
- Atomic upgrades and rollbacks

### npm/cargo/pip: Language Ecosystems

- Library and framework management
- Tight language integration
- Version resolution algorithms
- Large ecosystem for specific languages

## Conclusion

Zoi is not a replacement for existing package managers—it's a complement. Use Zoi when you need:

- Cross-platform consistency
- User-level package management  
- Multi-ecosystem integration
- Project environment management
- Custom package distribution

Combine Zoi with system package managers, language-specific tools, and containerization for a complete development workflow.
