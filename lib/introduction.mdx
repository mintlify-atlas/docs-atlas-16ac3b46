# Library API Introduction

Zoi provides a Rust library API that allows you to integrate its package management and environment setup capabilities directly into your Rust applications.

## What is Zoi Library?

The Zoi library (`zoi-rs`) exposes the core functionality of Zoi as a programmatic API. This enables you to:

- **Build packages** from `.pkg.lua` files programmatically
- **Install packages** from archive files into different scopes
- **Uninstall packages** with scope control
- **Manage package lifecycles** entirely from your Rust code

## Key Features

### Package Building
Build distributable `.pkg.tar.zst` archives from package definitions with support for multiple platforms and optional package signing.

### Package Installation
Install packages from local archives with fine-grained control over:
- Installation scope (User, System, or Project)
- Registry association
- Sub-package selection
- Conflict resolution

### Package Management
Full control over package lifecycles including uninstallation with scope-aware operations.

## Use Cases

- **Custom Package Managers**: Build specialized package managers on top of Zoi's infrastructure
- **CI/CD Pipelines**: Automate package building and deployment in continuous integration workflows
- **Development Tools**: Create development tools that manage dependencies programmatically
- **Testing Frameworks**: Test package installations and configurations automatically

## Public API Overview

The library exports three primary functions:

- `build()` - Build packages from `.pkg.lua` files
- `install_package()` - Install packages from archive files
- `uninstall_package()` - Remove installed packages

It also exports key types from the `pkg::types` module:

- `Scope` - Enum for package installation scopes (User, System, Project)
- Other package metadata types for advanced usage
