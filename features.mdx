Zoi is a universal package manager and environment setup tool designed to simplify package management and environment configuration across multiple operating systems.

## Core Features

<CardGroup cols={2}>
  <Card title="Cross-Platform Support" icon="computer">
    Works seamlessly on Linux, macOS, and Windows with a unified interface and experience.
  </Card>
  
  <Card title="Dynamic Package Definitions" icon="file-code">
    Define packages with dynamic Lua scripts (`.pkg.lua`) for maximum flexibility and power.
  </Card>
  
  <Card title="Universal Package Support" icon="box">
    Install packages from pre-built binaries, compressed archives, or build them from source.
  </Card>
  
  <Card title="Extensive Dependency Management" icon="sitemap">
    Integrates with over 40 system and language package managers (apt, brew, cargo, npm, pip, go, bun, etc.).
  </Card>
</CardGroup>

## Dependency Management

<CardGroup cols={2}>
  <Card title="Rich Dependencies" icon="link">
    Define runtime and build dependencies with required, optional, and selectable options.
  </Card>
  
  <Card title="Smart Provider Selection" icon="brain">
    Automatically resolve dependency conflicts with intelligent provider selection.
  </Card>
  
  <Card title="Virtual Packages" icon="ghost">
    Support for virtual packages (`provides`) and package replacement (`replaces`).
  </Card>
  
  <Card title="Multi-Manager Support" icon="layer-group">
    Pull dependencies from system managers (apt, pacman) and language-specific tools (npm, pip, cargo).
  </Card>
</CardGroup>

## Repository & Registry Features

<CardGroup cols={2}>
  <Card title="Repository-Based" icon="git-alt">
    Use official, community, or your own private/public Git-based repositories.
  </Card>
  
  <Card title="Multiple Registries" icon="database">
    Manage and sync from multiple package registries simultaneously.
  </Card>
  
  <Card title="Parallel Registry Sync" icon="sync">
    Concurrently sync multiple package registries for significantly faster updates.
  </Card>
  
  <Card title="Repository Tiers" icon="layer-group">
    Organize packages by stability: core, main, extra, community, test, archive, and zillowe.
  </Card>
</CardGroup>

## Security & Compliance

<CardGroup cols={2}>
  <Card title="Secure & Verifiable" icon="shield-check">
    Verifies package integrity with checksums and authenticity with GPG signatures.
  </Card>
  
  <Card title="Chain of Trust" icon="link">
    Git commit signature verification for entire registries with built-in PGP keyring.
  </Card>
  
  <Card title="Compliance & Audit Logging" icon="clipboard-list">
    Persistent, tamper-evident audit log of all package operations viewable via `zoi history`.
  </Card>
  
  <Card title="Enhanced Conflict Detection" icon="exclamation-triangle">
    Prevents packages from accidentally overwriting existing files with pre-download checking.
  </Card>
</CardGroup>

  <Card title="Collection" icon="boxes">

## Advanced CLI Features

Zoi provides a comprehensive set of CLI tools for package management:

| Command | Description |
|---------|-------------|
| `zoi history` | Read the persistent audit trail of past transactions |
| `zoi doctor` | Run system health checks and detect issues |
| `zoi why` | Understand why a package is installed |
| `zoi rollback` | Revert a package to its previous version or rollback the entire last transaction |
| `zoi pin` | Pin a package to a specific version |
| `zoi exec` | Run a package's binary without installing it |
| `zoi search -i` | Interactive TUI search with rich metadata |
| `zoi man` | View package documentation in the terminal |

## Library Support

Core functionality is available as a Rust library to be integrated into other applications. This allows developers to:

- Embed Zoi's package management capabilities into custom tools
- Build automation scripts with Rust
- Create custom package managers based on Zoi's architecture

See the [Library API documentation](/docs/zds/zoi/lib) for more information.
